# What is Acceptance Testing?
> Formal testing with respect to user needs, requirements, and business processes conducted to determine whether a system satisfies the acceptance criteria [3] and to enable the user, customers or other authorized entity to determine whether to accept the system [International Software Testing Qualifications Board]

# What is Acceptance Testing (Version 2)?
> A test level that focuses on determining whether to accept the system [International Software Testing Qualifications Board]

# User Acceptance Testing
* Process to verify that the solution works for the user (not system testing)
* AKA Beta testing
* Should be done by End User or Subject-Matter Expert (SME)
	* Preferably owner or client
* Consists of real-life scenario tests on system
* A final stage that occurs before client accepts new software
* A final verification of required functionality

# Operational Acceptance Testing
> Version 3: A type of acceptance testing performed to determine whether the organization responsible for operating the system can accept it. [International Software Testing Qualifications Board]
